## 脚本形式的网络结构

cccc的网络结构由一段C风格的脚本描述，在ini中，可以使用引号括起的多行字串。

下面是lenet的示例：

```c++
[cifa]
structure='
    batch=100;
    act = active_relu;
    X = Matrix(28, 28, 1, batch);
    A1 = active(maxpool(conv(X, Matrix(5, 5, 1, 50)) + Matrix(1, 1, 50, 1), { 2, 2 }), act);
    A2 = active(maxpool(conv(A1, Matrix(5, 5, 50, 50)) + Matrix(1, 1, 50, 1), { 2, 2 }), act);
    A = softmax_ce(Matrix(10, 256) * active(Matrix(256, A2.row()) * A2 + Matrix(256, 1), act) + Matrix(10, 1));
    setXY(X, A);
    addLoss(crossEntropy(A, getY()));
'
```

以上脚本并未进行计算，而是描述了一个网络的结构。

脚本中的可用函数请参考NetCifa.cpp中的实现。

常见的卷积、池化、激活函数：
```c++
B = conv(A, M)  //卷积操作，A为输入矩阵，M为卷积核矩阵
B = conv(A, M, {1, 1}, {1, 1})  //卷积操作，{1,1}为步长，{1,1}为填充大小（卷积算法由INI的[train]::conv_algo设置，默认自动选择）
B = maxpool(A, {2,2})  //最大池化操作，A为输入矩阵，{2,2}为池化窗口大小
B = averagepool(A, {2,2})  //平均池化操作，A为输入矩阵，{2,2}为池化窗口大小
B = reversepool(A, {2,2})  //反池化操作，A为输入矩阵，{2,2}为池化窗口大小，暂时只能平均反池化
B = elementmul(A, C)  //按元素相乘，A和C为输入矩阵
B = active(A, act)  //激活函数，A为输入矩阵，act为激活函数类型，如active_relu，active_sigmoid等

//激活也可以直接调用
B = softmax(A)  //对矩阵A的每一列进行softmax操作
B = sigmoid(A)  //对矩阵A的每个元素进行sigmoid操作
B = relu(A)  //对矩阵A的每个元素进行relu操作
```

乘法，加法，加偏置，均可以直接用运算符表示。

一些特殊函数：

```c++
setXY(X, A);  //指定输入输出项，除此二者外，中间的变量均可以复用

M(5, 5, 1, 50)  //表示创建一个5x5的卷积核，输入通道数为1，输出通道数为50

Matrix(28, 28, 1, batch)  //表示创建一个28x28的矩阵，通道数为1，batch大小为batch

getY()  //表示获取真实标签矩阵

getMatrix("name")  //获取名为name的矩阵，该矩阵在准备器中定义，由主流程负责传递给网络，可以用于较复杂的网络结构

addLoss(crossEntropy(A, getY()));  //损失函数，可有多个，此处表示使用A和真实标签计算交叉熵损失

```

如果需要调用ini中的其他定义，可以用section::key的形式。例如：

```c++
[train]
batch = 100
...

[cifa]
structure = '
    ...
    batch=train::batch
    ...
'
```

## 准备器

读取数据集需要用户自己编写C++代码实现，这部分会在主线程中运行。

用户需要编写从磁盘中读取数据到Matrix，以及一些自定义的变换等。基类中已经包含了一些简单变换，如亮度、对比度、翻转转置等。

如需要自己编写，需参考cccc-mnist的代码。

若设置mnist=1，则表示使用mnist的训练数据（60000个）。若为2，则表示使用测试数据（10000个）。
