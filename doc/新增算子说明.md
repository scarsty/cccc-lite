## 一、prependToken — 拼接 CLS token

**cifa 用法**
```c++
Y = prependToken(X, cls);
```

| 参数 | 形状 | 说明 |
|------|------|------|
| `X` | `(D, T, 1, B)` | 原始 patch/token 序列 |
| `cls` | `(D, 1, 1, 1)` | 可学习 CLS 向量，框架自动初始化为 0 并加入权重列表 |
| 返回 `Y` | `(D, T+1, 1, B)` | CLS 拼在头部，每个 batch 各复制一份 |

**说明**：正向把 `cls` 广播到所有 batch 并拼到 `X` 头部；反向把 dY 的第 0 个 token 梯度累加回 `cls.d()`，其余 token 梯度传回 `X.d()`。

**C++ 用法**
```cpp
// MatrixOp 层
MatrixOp op;
op.as_prependToken(X_sp, cls_sp, Y_sp);

// 直接调用 MatrixEx（无计算图时）——无单独 MatrixEx 接口，前向在 MatrixOp::forwardData() 内联实现
```

**C++ 注意事项**
- `X` 必须严格为 4 维 `(D, T, 1, B)`，否则 `as_prependToken` 输出 `LOG_ERR` 并不修改 `Y`。
- `cls` 的实际形状在 C++ 侧不做强制校验，约定传入 `(D, 1, 1, 1)`；cifa 注册时框架自动 `fillData(0)` 并加入 `weights_`，C++ 手动调用时需自行初始化并 `weights_.push_back(cls.get())`。
- 反向：`prependToken` 对 `dX` 使用**覆盖语义**（`keepWeight=0`），假设 `X` 在计算图中不被多处复用；若 `X` 有其他反向路径，需手动累加梯度。
- `dcls` 是所有 batch 对应位置梯度的**累加和**（`cls` 被广播，梯度需 reduce）。

## 二、firstToken — 取序列第 0 个 token

**cifa 用法**
```c++
Y = firstToken(X);
```

| 参数 | 形状 | 说明 |
|------|------|------|
| `X` | `(D, T, 1, B)` | token 序列 |
| 返回 `Y` | `(D, 1, 1, B)` | 每个 batch 的第 0 个 token |

**说明**：与 `prependToken` 配合实现 ViT 的 CLS-based 分类头（取 attention 后的 CLS 位置）。反向时 `dX[:,0,:,n] = dY`，其余 token 梯度清零。

**C++ 用法**
```cpp
MatrixOp op;
op.as_firstToken(X_sp, Y_sp);
```

**C++ 注意事项**
- `X` 必须严格为 4 维，否则输出 `LOG_ERR`。
- 反向**先 `fillData(0)` 清零整个 `dX`**，再写入第 0 个 token 的梯度。若 `dX` 已有其他路径的梯度（`keepWeight≠0`），会被覆盖——ViT 标准用法中 `X`（即 LayerNorm 的输出）为单路径，不受影响；若有残差连接到同一矩阵需注意。

**典型用法（ViT 分类头）**
```c++
cls_out = firstToken(layerNorm(R_last, lnw, lnb));
A = sigmoid_ce(Matrix(num_classes, 1, 1, D) * cls_out + Matrix(num_classes, 1, 1, 1));
```

## 三、rmsNorm — RMS Normalization

**cifa 用法**
```c++
Y = rmsNorm(X, scale);
Y = rmsNorm(X, scale, epsilon);  // 默认 epsilon=1e-6
```

| 参数 | 形状 | 说明 |
|------|------|------|
| `X` | 任意（`width_` 轴归一化） | 输入 |
| `scale` | `(width_, 1)` | 可学习增益，框架自动初始化为 1 并加入权重列表 |
| `epsilon` | 标量 | 数值稳定项，默认 `1e-6` |
| 返回 `Y` | 同 `X` | — |

**公式**：$y_i = \dfrac{x_i}{\sqrt{\frac{1}{n}\sum x_j^2 + \varepsilon}} \cdot s_i$

**与 layerNorm 的区别**：无均值项、无 bias，常用于 LLM（Qwen / LLaMA 等）。

**C++ 用法**
```cpp
// MatrixOp 层（推荐，自动管理 workspace）
MatrixOp op;
op.as_rmsNorm(X_sp, scale_sp, Y_sp);              // epsilon 默认 1e-6
op.as_rmsNorm(X_sp, scale_sp, Y_sp, 1e-5f);       // 自定义 epsilon

// MatrixEx 直接调用（跳过计算图时）
MatrixEx::rmsNormForward(X, Y, scale, epsilon);
MatrixEx::rmsNormBackward(X, Y, scale, epsilon);   // 反向前需已执行正向以填充 workspace
```

**内部状态 / Workspace**
- `Y.workspace_` 在首次 `rmsNormForward` 时懒初始化，共 2 项：
  - `[0]`：`invstd`，形状 `(outer,)`，`outer = (W*H*C/W) * N = total_elements / width_`
  - `[1]`：`dscale` 累加缓冲，形状 `(width_,)`，每次反向前自动 `fillData(0)`
- 归一化轴为 `width_`（最快变化维），`outer = row_ / width_ * number_`。

**C++ 注意事项**
- `scale` 形状必须为 `(width_, 1)` 或 `width_` 个元素，否则运行时访问越界。
- `scale` 需在 `weights_` 列表中（cifa 自动处理；C++ 手动用时调 `scale_sp->fillData(1)` 并 `weights_.push_back(scale_sp.get())`）。
- `rmsNormBackward` 把 `dscale` 写入 `Y.workspace_[1]`，调用者（`MatrixOp::backwardDataWeight`）再 `Matrix::add` 到 `scale->d()` 累加；直接用 `MatrixEx` 时需手动做这一步。

## 四、permute — 4 维任意轴置换

**cifa 用法**
```c++
Y = permute(X, {p0, p1, p2, p3});
```

| 参数 | 说明 |
|------|------|
| `X` | 任意 4 维矩阵，不足 4 维自动补 1 |
| `{p0,p1,p2,p3}` | 输出的第 i 维来自输入的第 `pi` 维 |
| 返回 `Y` | `dim_out[i] = dim_in[pi]` |

**示例**
```c++
// 交换 W(0) 与 C(2): (W,H,C,N) -> (C,H,W,N)
Y = permute(X, {2, 1, 0, 3});

// NCHW(W,C,1,N) -> (C,W,1,N)  (将头数和序列维交换, 用于 multi-head attention 重排)
Y = permute(X, {1, 0, 2, 3});
```

反向自动计算逆置换，调用同一个 CUDA kernel（`cuda_permute4d`）。

**C++ 用法**
```cpp
// MatrixOp 层
MatrixOp op;
op.as_permute(X_sp, Y_sp, {2, 1, 0, 3});

// MatrixEx 直接调用
MatrixEx::permute4dForward(X, Y, perm);
MatrixEx::permute4dBackward(X, Y, perm);   // perm 是正向 perm，内部自动求逆
```

**C++ 注意事项**
- `perm` 长度必须恰好为 4，否则 `as_permute` 输出 `LOG_ERR` 并退出。
- `X` 不足 4 维时，`as_permute` 会自动在后补 1（`dim_in.push_back(1)`）再做置换；`MatrixEx` 直接调用时需调用者保证 `Y` 已 resize 为正确形状。
- `perm` 以 `VectorAny` 存储在 `anys_[0]` 中，序列化/打印均为整数列表。
- 反向传入的仍是**正向 `perm`**，`permute4dBackward` 内部自动计算 `inv_perm` 然后调用同一 `cuda_permute4d` kernel，无需外部处理。

## 五、rope — 旋转位置编码（RoPE）

**cifa 用法**
```c++
Y = rope(X, cos_tab, sin_tab);
```

| 参数 | 形状 | 说明 |
|------|------|------|
| `X` | `(D, T, H, B)` | 需要编码的 Q 或 K，`D` 必须为偶数 |
| `cos_tab` | `(D/2, T, 1, 1)` | 预先计算好的 cos 表 |
| `sin_tab` | `(D/2, T, 1, 1)` | 预先计算好的 sin 表 |
| 返回 `Y` | 同 `X` | — |

**计算公式**（half-rotate，Qwen / LLaMA 风格）：
$$y_l = x_l \cos\theta - x_r \sin\theta, \quad y_r = x_r \cos\theta + x_l \sin\theta$$
其中 $l$ 为 $[0, D/2)$ 的下标，$r = l + D/2$。

**反向**：cos/sin 表视为常量，不回传梯度：
$$\frac{\partial L}{\partial x_l} = \frac{\partial L}{\partial y_l}\cos\theta + \frac{\partial L}{\partial y_r}\sin\theta, \quad
\frac{\partial L}{\partial x_r} = -\frac{\partial L}{\partial y_l}\sin\theta + \frac{\partial L}{\partial y_r}\cos\theta$$

**cos/sin 表的构造**（Python 参考，需导入为 Matrix 后传入）：
```python
# head_dim=D, max_seq=T, base=10000
theta = 1.0 / (base ** (np.arange(0, D, 2) / D))   # (D/2,)
pos   = np.arange(max_seq)                           # (T,)
freq  = np.outer(pos, theta)                         # (T, D/2)
# cos_tab.shape = (D/2, T), sin_tab.shape = (D/2, T)
cos_tab = freq.cos().T
sin_tab = freq.sin().T
```

**C++ 用法**
```cpp
// MatrixOp 层
MatrixOp op;
op.as_rope(X_sp, cos_tab_sp, sin_tab_sp, Y_sp);

// MatrixEx 直接调用
MatrixEx::ropeForward(X, Y, cos_tab, sin_tab);
MatrixEx::ropeBackward(X, Y, cos_tab, sin_tab);
```

**C++ 注意事项**
- `D = X.getWidth()` 必须为偶数，否则 `as_rope` 输出 `LOG_ERR` 并退出（`MatrixEx` 直接调用时不校验，需调用者保证）。
- `cos_tab / sin_tab` 约定形状 `(D/2, T, 1, 1)`，内存布局为列优先 `[i_dim, t_pos]`，即 `getData(t * half + i)` 取 pos=t、维度=i 的角度值。
- cos/sin 表**不回传梯度**（在 `backwardDataWeight` 中 `in_[1]`/`in_[2]` 不处理 needBack）；若要让 cos/sin 作为可训练参数，需修改反向逻辑。
- CPU fallback 与 CUDA kernel 结果一致；CUDA kernel 为 `cuda_rope_fwd` / `cuda_rope_bwd`，以 `(D, T, B)` 展开线程。

## 六、kvcache — KV Cache（推理专用）

**cifa 用法**
```c++
// 1. 预先分配持久缓冲（T_max 为最大生成长度）
k_cache = Matrix(D, T_max, H, B);
v_cache = Matrix(D, T_max, H, B);

// 2. 每步 forward: 写当前 token 的 K/V, 返回完整历史视图
K = kvcache(k_proj, k_cache);   // K 与 k_cache 共享内存，形状 = (D, T_max, H, B)
V = kvcache(v_proj, v_cache);
out = attention(Q, K, V);
```

| 参数 | 形状 | 说明 |
|------|------|------|
| `X_new` | `(D, T_new, H, B)` | 本步新增的 K 或 V |
| `cache` | `(D, T_max, H, B)` | 外部创建的持久缓冲矩阵 |
| 返回 `Y` | `(D, T_max, H, B)` | 与 `cache` 共享显存，可直接传给 attention |

**行为**：每次 `forward` 把 `X_new` 写入 `cache` 的当前 pos 偏移处，然后 pos 自动 +T_new。超出 T_max 的部分被丢弃（截断）。

**无反向**：仅用于推理（训练请直接传完整 K/V）。

**重置 pos**：在每段新的自回归序列开始前调用：
- C++ 侧：`net.resetKVCache()`
- cifa 脚本不提供此接口，需在外部调用

**C++ 用法**
```cpp
// MatrixOp 层（推荐）
MatrixOp op;
op.as_kvcache(x_new_sp, cache_sp, y_sp);
// Y 的 data 指针与 cache 共享（Y->shareData(*cache)）

// 重置当前 op 的写入位置
op.resetKVCache();

// 批量重置整个 op_queue 中所有 KV_CACHE 算子
MatrixOp::resetKVCache(op_queue);

// Net 便捷接口（重置 net.op_queue_ 内全部 KV cache）
net.resetKVCache();
```

**内部状态**
- 写入位置由 `MatrixOp::window_[0]`（当前 pos）维护，`window_[1]` 存储 `T_max`。
- `as_kvcache` 初始化 `window_ = {0, T_max}`；每次 `forwardData` 后 `window_[0] += T_new`。
- `resetKVCache()` 将 `window_[0]` 归零，`window_[1]` 不变。

**维度校验**（`as_kvcache` 构建时）
- `X_new.W == cache.W`（head_dim D 相同）
- `X_new.C == cache.C`（head 数 H 相同）
- `X_new.N == cache.N`（batch B 相同）
- `X_new.H <= cache.H`（新 token 数 ≤ T_max）
- 违反时输出 `LOG_ERR` 并提前返回（Y 不被修改）。

**注意事项**
- Y 与 cache **共享显存指针**（`Y->shareData(*cache)`），不另分配；对 Y 的读操作等同于读 cache。
- `cache` 矩阵应在推理开始前 `fillData(0)` 清零，防止未填充区域包含历史垃圾数据影响 attention 分数。
- 超出 `T_max` 的写入被**静默截断**（不报错），调用者应自行控制 pos 不超界。
- 下游 `attention(Q, K, V)` 对 `T_max` 个位置全部计算，需要配合 causal/length attention mask 屏蔽未填充的尾部。当前 `attention` 算子暂不支持 mask，完整 LLM 推理需额外处理。
- 训练阶段请勿使用此算子（无反向，backwardDataWeight 为 no-op）。

## 七、pixelShuffle — 亚像素上采样（Sub-pixel / Pixel Shuffle）

**cifa 用法**
```c++
Y = pixelShuffle(X, r);
```

| 参数 | 形状 | 说明 |
|------|------|------|
| `X` | `(W, H, C_out*r*r, N)` | 输入，通道数必须是 `r*r` 的整数倍 |
| `r` | 整数标量 | 上采样因子（upscale factor） |
| 返回 `Y` | `(W*r, H*r, C_out, N)` | 空间分辨率扩大 r 倍，通道数缩小 r² 倍 |

**映射关系**
$$Y[x \cdot r + dx,\; y \cdot r + dy,\; c,\; n] = X[x,\; y,\; c \cdot r^2 + dy \cdot r + dx,\; n]$$

即：输入的 `r*r` 个通道"折叠"到输出的 `r×r` 空间子格里，每个 token 的值映射到空间上连续的一个 patch，无跨 patch 混淆。

**反向**：同映射的逆操作：
$$dX[x, y, c \cdot r^2 + dy \cdot r + dx, n] = dY[x \cdot r + dx,\; y \cdot r + dy,\; c,\; n]$$

**典型用法（Transformer 解码器，无 CNN 上采样）**
```c++
D = 64;
Nfc  = reshape(Nf, {8, 8, D, batch});          // (8,8,64,B) — bottleneck tokens

P1in = active(conv(Nfc, Matrix(1,1,D,256)) + Matrix(1,1,256,1), act);
P1   = pixelShuffle(P1in, 2);                   // (16,16,64,B)

P2in = active(conv(P1, Matrix(1,1,64,256)) + Matrix(1,1,256,1), act);
P2   = pixelShuffle(P2in, 2);                   // (32,32,64,B)

P3in = active(conv(P2, Matrix(1,1,64,256)) + Matrix(1,1,256,1), act);
P3   = pixelShuffle(P3in, 2);                   // (64,64,64,B)

P4in = active(conv(P3, Matrix(1,1,64,256)) + Matrix(1,1,256,1), act);
P4   = pixelShuffle(P4in, 2);                   // (128,128,64,B)

A = sigmoid_ce(conv(P4, Matrix(1,1,64,1)) + Matrix(1,1,1,1));
```

**与 reversePool + 3×3 conv 解码器的对比**

| 方案 | 空间信息 | 跨 patch 交互 | 典型 Error |
|---|---|---|---|
| reversePool + 3×3 conv | 好（3×3 感受野跨 patch） | 有 | 2.19 |
| pixelShuffle + 1×1 conv | 每 token 独立，无跨 patch | 无 | 23.58 |
| 1×1 conv 解码（无 shuffle） | 无空间感知 | 无 | 492 |

**C++ 用法**
```cpp
// MatrixOp 层
MatrixOp op;
op.as_pixelShuffle(X_sp, Y_sp, r);

// MatrixEx 直接调用
MatrixEx::pixelShuffleForward(X, Y, r);
MatrixEx::pixelShuffleBackward(X, Y, r);   // X 接收梯度，Y 存有正向结果
```

**C++ 注意事项**
- `X.channel()` 必须是 `r*r` 的整数倍，否则 `as_pixelShuffle` 输出 `LOG_ERR` 并退出。
- `Y` 的形状由 `as_pixelShuffle` 自动设置为 `{W*r, H*r, C_in/(r*r), N}`，调用者无需预先 resize。
- `r` 以 `window_[0]` 存储；打印格式为 `Y = pixelShuffle(X, r)`。
- CUDA kernel：`cuda_pixel_shuffle_fwd` / `cuda_pixel_shuffle_bwd`，线程以 `(W*r, H*r, C_out, N)` 展开。
- CPU fallback 与 CUDA 结果一致（逐元素映射，无近似）。

