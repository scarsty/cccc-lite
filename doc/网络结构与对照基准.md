# cccc 网络结构与对照基准

本文记录 `work/` 目录下对照训练任务的网络结构、训练超参与实测结果。

- CNN 基准：[work/cnn-compare.ini](../work/cnn-compare.ini)（经典 LeNet 风格，源自 `work/mnist-lenet.ini`）
- Transformer 基准：[work/transformer-test.ini](../work/transformer-test.ini)（真 self-attention ViT，使用本仓库新增的 `batchedMul` 算子）
- will-extension2 CNN：[work/will-cnn.ini](../work/will-cnn.ini)（移植自 `will-extension2/NetString.cpp`）

各网络均使用 `cccc-mnist` 数据集，相同 128×128 输入，10 类分类。

## 1. 共同训练配置

| 项 | 取值 |
| --- | --- |
| solver | sgd |
| momentum | 0.9 |
| weight_decay | 5e-4 |
| batch | 100 |
| train_epochs | 10 |
| test_epoch | 1 |
| OutIter | 200 |
| 数据集 | MNIST（cccc-mnist 插件） |
| 模式 | Release（`x64\Release\cccc-windows.exe`） |

只有以下两项按各自网络的"原生"尺度调过：

| 项 | LeNet CNN | ViT |
| --- | --- | --- |
| 输入分辨率 | 128×128（`resize_to=128`） | 128×128（`resize_to=128`） |
| learn_rate_base | 1e-2 | 3e-3 |
| init_weight | 默认 | msra |
| stop_no_effective_grad | 0 | 1 |

学习率不同的原因：ViT 含 LayerNorm + 多层 1×1 conv + batchedMul，整体梯度尺度较小，3e-3 合适；CNN 结构简单、无归一化层，1e-2 才能在前两个 epoch 建立有效梯度。

> 注：CNN 需关闭 `stop_no_effective_grad`（设为 0）。sigmoid_ce 对 10 类输出按独立二分类评估，各类单独准确率在前 2 epoch 可能低于 10% 阈值，被框架误判为"未建立有效梯度"并提前终止。ViT 因 LayerNorm 平滑梯度，epoch 1 即突破该阈值。

## 2. 网络结构

### 2.1 CNN 基准（LeNet 风格）

cifa 结构（[work/cnn-compare.ini](../work/cnn-compare.ini)）：

```text
X    = Matrix(128, 128, 1, batch);                              # 输入
C1   = relu(maxpool(conv(X,  5x5x1x32)  + bias, {2,2}));       # 128 -> 64, 通道 32
C2   = relu(maxpool(conv(C1, 5x5x32x64) + bias, {2,2}));       # 64  -> 32, 通道 64
C3   = relu(maxpool(conv(C2, 3x3x64x64) + bias, {2,2}));       # 32  -> 16, 通道 64
C4   = relu(maxpool(conv(C3, 3x3x64x128)+ bias, {2,2}));       # 16  ->  8, 通道 128
Pool = averagePool(C4, {8,8}, {8,8});                          # GAP -> (1,1,128,B)
F1   = relu(Matrix(256, 1, 1, 128) * Pool + bias);             # FC -> 256
A    = sigmoid_ce(Matrix(10, 1, 1, 256) * F1 + bias);          # FC -> 10
```

要点：
- 4 段 conv-pool 将 128×128 下采样到 8×8，再 GAP → 128 维特征向量。
- FC 权重写成四维 `Matrix(out, 1, 1, in)`，避免二维形式的 `width_=1` 塌缩陷阱。
- 末端 `sigmoid_ce` 已包含 loss，不再额外 `addLoss`。

### 2.2 Transformer ViT（真 self-attention）

cifa 结构（[work/transformer-test.ini](../work/transformer-test.ini)，要点）：

```text
D = 64;  T = 64;  batch = 100;
X      = Matrix(128, 128, 1, batch);

# patch embedding: 128/16 = 8, 共 8x8=64 个 token, 每 token 维度 D=64
PE2D0  = reshape(conv(X, 16x16x1xD, stride={16,16}) + bias, {D, T, 1, batch});

# 每个 transformer block
H1     = layerNorm(PE2D0, ln1w, ln1b);                  # pre-LN
H1c    = reshape(H1, {8, 8, D, batch});                 # token 维度搬到通道, 保留 batch
Q1c    = conv(H1c, 1x1xDxD);  K1c = ...;  V1c = ...;    # 1x1 conv 实现 token-wise 线性投影 Wq/Wk/Wv
Q1     = reshape(Q1c, {D, T, 1, batch});                # 还原成 (D, T, 1, B)
A1     = attention(Q1, K1, V1, D);                      # 内部用 batchedMul
A1c    = reshape(A1, {8, 8, D, batch});
O1     = reshape(conv(A1c, 1x1xDxD), {D, T, 1, batch}); # Wo 投影
R1     = PE2D0 + O1;                                    # 残差

F1in   = layerNorm(R1, ln1bw, ln1bb);                   # pre-LN
F1conv = reshape(F1in, {8, 8, D, batch});
F1a    = relu(conv(F1conv, 1x1xDx256) + bias);          # FFN 升维 (1x1 conv 等价于 token-wise FC)
F1     = reshape(conv(F1a, 1x1x256xD) + bias, {D, T, 1, batch});
R1b    = R1 + F1;                                       # 残差

# 第二个 block 与上同 (R1b -> R2b)

N3     = layerNorm(R2b, ln3w, ln3b);
N3conv = reshape(N3, {8, 8, D, batch});
Pool   = averagePool(N3conv, {8, 8}, {8, 8});           # GAP -> (1,1,D,B)
A      = sigmoid_ce(Matrix(10, 1, 1, D) * Pool + bias); # 分类头
```

新增的关键算子（本次任务实现）：

- `batchedMul(A, B, M, N, K, batch, [ta, tb])`：对每个 batch slice 做 col-major GEMM `R = A · B`，CUDA 路径直接调用 `cublasSgemmStridedBatched`，HIP/CPU 退化为逐 batch 调用。
  - 实现：[Matrix::mulBatched](../cccc/Matrix.cpp)、[MatrixOpType::BATCHED_MUL](../cccc/MatrixOp.cpp)、cifa 注册见 [NetCifa.cpp](../cccc/NetCifa.cpp)。
- `attention(Q, K, V, dk)` 在 cifa 中已重写为 `K^T·Q → /√dk → softmax_channel → V·attn`，全程使用 `batchedMul`，是真正的 self-attention（而不是早期占位实现）。

> **设计注（Q/K/V 投影为何用 1×1 conv）**：直觉写法是把 `(D, T, 1, B)` reshape 成 `(D, 1, 1, T·B)` 再用 `mul` 投影，但 `as_reshape` 强制 `dim.back()=B`，不能跨 batch 维度 reshape，否则反向会触发 `Memcpy error 1`。因此改为 `(D, T, 1, B) → (8, 8, D, B)` 把 token 维并入通道，再用 1×1 conv 实现 token-wise 线性投影，形状始终保留 batch 维。

### 2.3 will-extension2 CNN

参照 `will-extension2/NetString.cpp` 第 68 行起的 y_type=0/5 分类路径移植，配置文件：[work/will-cnn.ini](../work/will-cnn.ini)。

- `conv_c = 50`（通道数）
- `conv_r = 3`（等于 `log2(128) - 4`，控制中间池化层数）
- 首层之后均为 3×3 卷积（stride=1, pad=1）+ maxpool(2×2)，最终 feature map 4×4×50 = 800 维，接三层 FC（800→128→128→10）。

**full_scan = 0**（首层 stride=2，激进下采样）：

```
X  (128×128×1)
└─ conv(5×5, stride={2,2}, pad={2,2}) + maxpool(2×2) → 32×32×50
└─ conv(3×3, stride={1,1}, pad={1,1}) + maxpool(2×2) → 16×16×50   (loop i=0)
└─ conv(3×3, stride={1,1}, pad={1,1}) + maxpool(2×2) → 8×8×50     (loop i=1)
└─ conv(3×3, stride={1,1}, pad={1,1}) + maxpool(2×2) → 4×4×50  = A1
└─ FC(800→128) → relu → FC(128→128) → relu → FC(128→10) → sigmoid_ce
```

**full_scan = 1**（首层 stride=1，保留细节）：

```
X  (128×128×1)
└─ conv(5×5, stride={1,1}, pad={2,2}) + maxpool(2×2) → 64×64×50
└─ conv(3×3, stride={1,1}, pad={1,1}) + maxpool(2×2) → 32×32×50   (if 分支额外层)
└─ conv(3×3, stride={1,1}, pad={1,1}) + maxpool(2×2) → 16×16×50   (loop i=0)
└─ conv(3×3, stride={1,1}, pad={1,1}) + maxpool(2×2) → 8×8×50     (loop i=1)
└─ conv(3×3, stride={1,1}, pad={1,1}) + maxpool(2×2) → 4×4×50  = A1
└─ FC(800→128) → relu → FC(128→128) → relu → FC(128→10) → sigmoid_ce
```

两者最终 feature map 尺寸相同；full_scan=1 多一次 64×64 级别的卷积，计算量明显更大。

cifa 脚本使用 `if/else` 和 C 风格 `for` 循环动态控制层结构：
```c
if (full_scan) { /* stride=1 路径，多一层 */ }
else           { /* stride=2 路径 */ }
for (i = 0; i < conv_r - 1; i++) { /* 中间 conv+pool 层 */ }
```

## 3. ViT 分类头设计

为研究 ViT 末端"如何聚合 token 序列做分类"对精度的影响，在同一 2 block ViT 主干（D=64, FFN=256, T=64 patches）上替换最后的分类头，得到 4 个变体。除分类头与必要的尾部算子外，其余结构、超参、数据增强、随机种子（默认）一致。

### 3.1 各变体对比结果

| 变体 | 配置文件 | 分类头 | 最佳测试准确率 | 10 epoch 总耗时 |
| --- | --- | --- | --- | --- |
| GAP（基线） | `transformer-test.ini` | `averagePool({8,8}) → FC(D→10)` | **96.45%** | **33.77 s** |
| flat FC | `transformer-fc.ini` | `reshape(N3,{8,8,D,B})` 后直接 `FC(8·8·D→10)` | **98.45%** | **39.99 s** |
| 学习权重池 | `transformer-attnpool.ini` | `conv(N3, {1,T,1,1})` 学习按位置加权求和 | **96.63%** | **41.26 s** |
| **真 CLS token** | `transformer-cls.ini` | `prependToken(patches, cls)` + `firstToken(N3)` + `FC(D→10)` | **97.89%** | **95.21 s** |

为支持真 CLS token，本次新增了两个算子：

- `prependToken(X, cls)`：把可学习的 `(D,1,1,1)` cls 向量广播到 batch 的每个样本头部，将 `(D,T,1,B)` 拼成 `(D,T+1,1,B)`。前向逐 batch 用 `Matrix::copyDataPtr` 写入；反向 `dX = dY[:, 1:T+1, :, :]`，`dcls = Σ_n dY[:, 0, :, n]`（用单 sample `(D,1,1,1)` 视图通过 `Matrix::add` 累加）。
- `firstToken(X)`：从 `(D,T,1,B)` 中取每个 batch 的第 0 个 token，得 `(D,1,1,B)`。反向只把 `dY` 写到 `dX` 的第 0 个 token 槽位，其余清零。

实现位置：`cccc/MatrixOp.h/.cpp`（`PREPEND_TOKEN`、`FIRST_TOKEN`），cifa 注册见 `cccc/NetCifa.cpp`。

> 引入 CLS 后序列长度变成 T+1=65，无法像基线那样 `reshape(.., {8, 8, D, batch})`（8×8=64）。改用 `reshape(.., {5, 13, D, batch})`，5×13=65，总元素数与 batch 末位均保留。

观察：

1. **flat FC 仍是最强的头**（98.45%）：直接把整个 token 序列展平接 FC，相当于把 8×8×D = 4096 维当向量分类；在 MNIST 这种简单任务上，更多参数 + 全部位置同等可见 = 更高精度。
2. **真 CLS（97.89%）显著优于 GAP（96.45%）和学习权重池（96.63%）**，证明"先让 attention 把信息聚合到一个专门的 token，再用单 token 分类"确实带来增益（+1.4 pp）。学习权重池只学了"位置权重"，无法基于内容动态聚合；CLS 通过两层 self-attention 让所有 patch 都能贡献到第 0 个 token，是真正的内容驱动聚合。
3. **CLS 较慢（95s ≈ 2.4× GAP 的 33s）**：原因是算子前向/反向按 batch 循环做小块 `cudaMemcpy`，未做 batched 实现；后续可改为单次 strided/批量拷贝。
4. **真 CLS 仍略输 flat FC**（97.89% vs 98.45%）：对 MNIST 而言，"暴力扁平接 FC"的归纳偏置正好契合，CLS 反而被 attention 路径的瓶颈限制。CLS 的优势主要体现在大规模预训练 + 下游分类的场景。

## 4. 汇总基准结果（Release 模式）

实测设备：当前主机 GPU（`gpu=1`，cuDNN/cublas 动态加载）。

| 指标 | CNN (LeNet 风格) | ViT real attention | will-CNN fs=0 | will-CNN fs=1 |
| --- | --- | --- | --- | --- |
| 配置文件 | `cnn-compare.ini` | `transformer-test.ini` | `will-cnn.ini`（fs=0） | `will-cnn.ini`（fs=1） |
| 首层 conv 步长 | 2 | — | 2 | 1 |
| 总层数 | 4 conv + FC | 2 block attention | conv_r+1 = 4 conv + FC | conv_r+2 = 5 conv + FC |
| 参数规模 | ~5.6 M | ~250 K | ~600 K | ~600 K |
| 10 epoch 总耗时 | **58.86 s** | **33.77 s** | **28.65 s** | **76.56 s** |
| 最佳测试准确率 | **98.63%** | **96.45%** | **99.03%** | **99.19%** |

观察：

1. **速度**：CNN 比 ViT 慢 ~1.7×（58.86s vs 33.77s）。CNN 的慢来自 4 层大核 conv 在大 feature map 上的滑动计算；ViT 的 1×1 conv 本质是矩阵乘法，cuDNN 调度效率更高。
2. **准确率**：CNN 高 ~2.2 pp（98.63% vs 96.45%）。MNIST 是局部纹理任务，CNN 的归纳偏置天然适合；ViT 以 ~250K 参数（CNN 的 1/22）取得接近的效果，体现了 self-attention 的参数效率。
3. **收敛**：ViT 因 LayerNorm 在 epoch 1 就达到 50%+，收敛平滑；CNN 前两个 epoch 在随机水平徘徊，epoch 3 才突破 86%。
4. **will-CNN fs=0 最快且精度最高**（28.65s / 99.03%）。首层大步长减少后续计算量，50 通道专注于中频特征，对 MNIST 有效。
5. **full_scan=1 显著变慢**（76.56s ≈ 2.7× fs=0），准确率仅微升 0.16 pp，性价比低。
6. **will-CNN vs LeNet CNN**：will-CNN 通道更少但池化更密，速度快 ~2×，准确率高 ~0.4 pp，密集池化 + 浅宽结构对 MNIST 有优势。
7. **batchedMul 正确性**：ViT 所有 10 epoch 无 weight rollback，Q/K/V/Wo 四组梯度全部正常回传，端到端收敛验证通过。

## 6. MNIST 128×128 显存分析

将 MNIST 上采样到 128×128（`resize_to_=128`）后，训练时会遇到显存压力。以下是实测结论。

### 6.1 batch 大小对显存的影响（几乎可忽略）

对于 CNN，每个 batch 的中间激活量与 batch size 成正比，但幅度远小于数据集总量。实测 32GB GPU：

- `batch=10`：CNN 层激活约 +7 MiB（128²×1×50ch×32 layer×10B）
- `batch=100`：约 +70 MiB

相比几 GB 的数据集显存，batch 大小的影响不超过 3%。**调整 batch 无法解决显存不足问题**。

### 6.2 显存的真正大头：fill_group 数据预加载

`fill_group` 控制一次性预加载到显存的样本数。MNIST 128×128（灰度）：

$$\text{fill\_group} \times 128^2 \times 1 \times 4\,\text{B} = \text{fill\_group} \times 65{,}536\,\text{B}$$

例：`fill_group=5000` → $5000 \times 65{,}536 \approx 313\,\text{MB}$（输入 $X$）。自编码器同时存 $Y$，则约 626 MB。

若不设 `fill_group`（全量上传）：

$$60{,}000 \times 128^2 \times 1 \times 4\,\text{B} \approx 3.75\,\text{GB}$$

自编码器同时存 $X$ 和 $Y$：$3.75 + 3.75 = 7.5\,\text{GB}$，常规 GPU 直接 OOM。

**结论**：`fill_group` 是控制训练数据显存占用的核心参数，必须与 GPU 显存容量匹配。

### 6.3 cuDNN workspace 是另一个大头

cccc 调用 cuDNN 卷积时，cuDNN 会为每层卷积分配 workspace（算法测试缓冲区）并缓存在 handle 中。累计量随层数增长：

- **CNN 3 层自编码器**（128×128）：约 **2.6 GB**
- **Transformer 自编码器**（128×128，多层）：约 **21 GB**（含网络参数；见 6.4 节实测数据）

这部分显存在 cuDNN handle 生命周期内持续占用，是仅次于数据的第二大开销。cccc 已将算法选择切换为 `Get*_v7`（启发式，无额外 workspace），详见架构说明文档中的 cuDNN workspace 分析。

### 6.4 Transformer 自编码器全量加载的实测数据

在 32 GB GPU 上，Transformer 自编码器（128×128 MNIST，无 fill_group）实测：

- 数据预加载（X + Y 全量）：**28,444 MiB ≈ 27.8 GB**
- 随后第一次前向传播：OOM（剩余约 4 GB 不足以容纳 cuDNN workspace + 层激活）

设置 `fill_group=5000` 后，数据预加载降至 ≈ 313 MB，训练正常进行。
