# cccc 网络脚本语言说明

cccc 的网络结构由一段 C 风格脚本描述，写在 INI 配置文件的 `[cifa]` 段中。脚本本身并不执行计算，而是**描述计算图的结构**，由框架在训练/推理时根据该结构依次执行前向和反向传播。

---

## 一、写在 INI 中的形式

`[cifa]` 段支持两个键：
- `parameter`：先于 `structure` 执行，通常用于声明超参数
- `structure`：网络结构主脚本

脚本用单引号括起，支持多行字符串：

```ini
[cifa]
parameter = '
    batch = 100;
    lr    = 1e-3;
'
structure = '
    X = Matrix(28, 28, 1, batch);
    ...
    setXY(X, A);
    addLoss(crossEntropy(A, getY()));
'
```

脚本中可以用 `section::key` 的方式引用 INI 其他段的值：

```c++
batch = train::batch;
lr    = train::learn_rate_base;
```

cifa 支持 C 风格的 `auto` 变量声明、`for`/`if` 控制流以及 `{v1, v2, ...}` 列表字面量：

```c++
auto batch = 100;
auto act   = active_relu;
Y = maxPool(X, {2, 2});   // {2,2} 即 [2,2] 列表
for (i = 0; i < 10; i++) { ... }
```

## 二、矩阵创建

| 函数 | 说明 |
|------|------|
| `Matrix(w, h, c, n)` | 创建矩阵，参数可为 1–4 个维度值（w、h、c、n），不足的维度默认为 1 |
| `M(w, h, c, n)` | 同上，`Matrix` 的简写 |

矩阵的四个维度依次为：**宽(w) × 高(h) × 通道(c) × 批次(n)**。

> 通过 `Matrix()` 创建的矩阵默认允许反向传播和参数加载。  
> 框架会根据计算图拓扑自动识别权重矩阵（不连接 X、但连接到损失函数的矩阵），通常无需手动指定。

```c++
X  = Matrix(28, 28, 1, batch);   // 输入：28×28 灰度图，batch 张
W1 = Matrix(5, 5, 1, 32);        // 卷积核：5×5，1 输入通道，32 输出通道
b1 = Matrix(1, 1, 32, 1);        // 偏置：每通道 1 个标量
Wfc = Matrix(256, 800);          // 全连接权重：256 输出，800 输入（参数可少于 4 个）
```

## 三、基本运算符

矩阵对象支持直接使用运算符：

| 运算符 | 含义 | 自动判断 |
|--------|------|----------|
| `A + B` | 矩阵加法 | 若 B 的 number=1 且尺寸与 A 不同，自动视为加偏置（addBias） |
| `A * B` | 矩阵乘法（gemm） | — |
| `A * k` / `k * A` | 标量缩放 | k 为数值 |

## 四、网络层函数

### 卷积 / 相关

```c++
Y = conv(X, W)                    // 卷积，步长默认 {1,1}，填充默认 {0,0}
Y = conv(X, W, stride, padding)   // 指定步长和填充，例如 {1,1}、{0,0}
Y = corr(X, W, stride, padding)   // 互相关（不翻转卷积核）
```

> 卷积算法由 INI `[train]` 段的 `conv_algo` 键控制（默认 -1 自动选择），不在脚本中指定。

### 池化

```c++
Y = maxPool(X, window)                       // 最大池化
Y = maxPool(X, window, stride, padding)
Y = averagePool(X, window)                   // 平均池化（不含 padding）
Y = averagePool(X, window, stride, padding)
Y = reversePool(X, window)                   // 反池化（平均反池化）
Y = reversePool(X, window, stride, padding)
Y = pool(X, pool_type, window, stride, padding)  // 通用池化，pool_type 见下
Y = poolChannel(X, pool_type)                // 在通道维度上池化
```

`pool_type` 常量（可用于 `pool()` 第二参数，同时也注册为数值常量）：

| 常量 | 说明 |
|------|------|
| `pool_max` | 最大池化 |
| `pool_average` | 平均池化（不含 padding，`pool_average_no_padding` 的别名） |
| `pool_average_no_padding` | 平均池化（不含 padding） |
| `pool_average_padding` | 平均池化（含 padding） |
| `pool_ma` | 实验性池化 |

### 激活函数

```c++
Y = active(X, act_type)              // 通用激活，act_type 见下（大小写不敏感）
Y = active(X, act_type, int_params)  // 含整数参数（如 clipped_relu 的上限）
// 各激活函数也可直接按名称调用：
Y = relu(X)               Y = sigmoid(X)
Y = tanh(X)               Y = softmax(X)
Y = softmax_fast(X)       Y = softmax_log(X)
Y = sigmoid_ce(X)         Y = softmax_ce(X)    // CE：反向误差直通，用于多出口网络
Y = softmax_fast_ce(X)    Y = elu(X)
Y = selu(X)               Y = silu(X)
Y = leaky_relu(X)         Y = dropout(X)
Y = abs(X)                Y = square(X)
Y = sin(X)                Y = zigzag(X)
Y = sum_max(X)            Y = zero_channel(X)
Y = softplus(X)           // 仅 CPU
```

`act_type` 常量完整列表（由 Option 注册，脚本中通过 `active_<name>` 引用，数值仅供参考）：

| 常量 | 值 | 含义 |
|------|----|------|
| `active_none` | -1 | 无激活 |
| `active_sigmoid` | 0 | Sigmoid |
| `active_relu` | 1 | ReLU |
| `active_tanh` | 2 | Tanh |
| `active_clipped_relu` | 3 | Clipped ReLU |
| `active_elu` | 4 | ELU（仅 GPU） |
| `active_softmax` | 5 | Softmax |
| `active_softmax_fast` | 6 | Softmax Fast |
| `active_softmax_log` | 7 | Softmax Log |
| `active_dropout` | 9 | Dropout |
| `active_softplus` | 11 | Softplus（仅 CPU） |
| `active_sigmoid_ce` | 20 | Sigmoid + CE 直通 |
| `active_softmax_ce` | 21 | Softmax + CE 直通 |
| `active_softmax_fast_ce` | 22 | Softmax Fast + CE 直通 |
| `active_sin` | 23 | Sin |
| `active_zigzag` | 24 | Zigzag |
| `active_leaky_relu` | 26 | Leaky ReLU（别名 `active_lrelu`） |
| `active_selu` | 27 | SELU |
| `active_abs` | 28 | Abs |
| `active_silu` | 30 | SiLU |

### 其他层

```c++
Y = add(A, B)                  // 矩阵加法（等价于 A + B）
Y = add([A, B, C], [w1,w2,w3]) // 多输入加权求和
Y = addBias(X, b)              // 显式加偏置
Y = elementMul(A, B)           // 逐元素乘法
Y = mul(A, B)                  // 矩阵乘法（可指定维度参数）
Y = concat([A, B, C])          // 通道维拼接
Y = reshape(X, dim)            // 改变形状，dim 为整数列表
Y = max(A, B)                  // 逐元素取较大值
Y = scale(X, k)                // 标量缩放（等价于 X * k）
```

## 六、损失函数

```c++
addLoss(crossEntropy(A, getY()))           // 交叉熵损失
addLoss(crossEntropy(A, getY(), weight))   // 带权重的交叉熵
addLoss(focal(A, getY()))                  // Focal Loss
addLoss(L2(W))                             // L2 正则化（权重衰减）
addLoss(0.5 * crossEntropy(A, getY()) + 0.5 * L2(W))  // 多损失加权组合
```

## 七、网络输入输出绑定

```c++
setXY(X, A);   // 指定输入矩阵 X 和网络预测输出矩阵 A（必须调用）
setXA(X, A);   // 同上（别名）
```

> `setXY(X, A)` 中的第二个参数 `A` 是**网络预测输出**（计算图末端激活的结果），而非标签。

获取外部传入矩阵：

```c++
X = getX()             // 获取输入矩阵（由数据准备器填充，对应 setXY 的第一个参数）
A = getA()             // 获取网络预测输出矩阵（对应 setXY 的第二个参数，用于外部读取结果）
Y = getY()             // 获取标签矩阵（由数据准备器在训练时填充的真实标签，用于计算损失）
lw = getLossWeight()   // 获取损失权重矩阵（由 setLossWeight 设置）
m  = getMatrix("name") // 获取由准备器传入的命名矩阵
ok = haveMatrix("name")// 判断该矩阵是否存在
showMatrix("name", M)  // 将矩阵注册为可被外部访问的命名矩阵
```

> 在计算损失时，通常写 `crossEntropy(A, getY())`：`A` 是预测，`getY()` 是训练标签。

## 八、权重管理

```c++
addWeight(W1, W2, ...)   // 将矩阵提前加入权重列表，使其跳过 initWeights 随机重初始化（保留原始值）
clearWeight()            // 清空手动添加的条目（不影响自动识别逻辑）
```

> **正常情况下无需调用 `addWeight()`**。  
> 框架在 `initWeights()` 中会自动识别所有结构性权重并进行随机初始化。  
> `addWeight(W)` 的主要用途是：将矩阵提前写入权重列表，这样 `initWeights()` 会跳过对它的随机重初始化（例如已通过 `setValue` 手动赋值，或将要从已保存模型加载的矩阵）。  
> 最终 `weights_` 列表中会同时包含手动添加和自动识别的所有权重，一起由 Solver 更新并进行模型保存/读取。

## 九、控制训练行为的状态开关

```c++
setNeedBackState(1)   // 后续创建的矩阵默认参与反向传播（默认 1）
setNeedBackState(0)   // 后续创建的矩阵不参与反向传播（用于冻结层）
setNeedLoadState(1)   // 后续创建的矩阵在加载模型时读取已保存参数（默认 1）
setNeedLoadState(0)   // 不加载（用于全新初始化的层）

setNeedBack(M, 0)     // 单独设置某个矩阵是否反向传播
setAllNeedBack(0)     // 对已注册的所有矩阵统一设置

setCurrentSolverType("adam")  // 设置后续层使用的求解器类型
```

## 十一、完整示例：LeNet

```ini
[cifa]
structure = '
    auto batch = 100;
    auto act   = active_relu;

    auto X  = Matrix(28, 28, 1, batch);

    // 第一卷积块
    auto A1 = active(maxPool(conv(X, Matrix(5, 5, 1, 32)) + Matrix(1, 1, 32, 1), {2, 2}), act);

    // 第二卷积块
    auto A2 = active(maxPool(conv(A1, Matrix(5, 5, 32, 64)) + Matrix(1, 1, 64, 1), {2, 2}), act);

    // 全连接层
    auto fc1 = active(Matrix(256, A2.row()) * A2 + Matrix(256, 1), act);
    auto A   = softmax_ce(Matrix(10, 256) * fc1 + Matrix(10, 1));

    setXY(X, A);
    addLoss(crossEntropy(A, getY()));
'
```

## 十二、示例：带冻结层的迁移学习

```ini
structure = '
    auto batch = 32;
    auto X = Matrix(224, 224, 3, batch);

    // 冻结的特征提取部分（不更新权重，但加载已保存参数）
    setNeedBackState(0);
    setNeedLoadState(1);
    auto F = relu(conv(X, Matrix(3, 3, 3, 64)) + Matrix(1, 1, 64, 1));
    // ... 更多冻结层

    // 可训练的分类头（从头初始化，不加载旧参数）
    setNeedBackState(1);
    setNeedLoadState(0);
    auto A = softmax_ce(Matrix(10, F.row()) * F + Matrix(10, 1));

    setXY(X, A);
    addLoss(crossEntropy(A, getY()));
'
```
