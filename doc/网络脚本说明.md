# cccc 网络脚本语言说明

cccc 的网络结构由一段 C 风格脚本描述，写在 INI 配置文件的 `[cifa]` 段中。脚本本身并不执行计算，而是**描述计算图的结构**，由框架在训练/推理时根据该结构依次执行前向和反向传播。

---

## 一、写在 INI 中的形式

`[cifa]` 段支持两个键：
- `parameter`：先于 `structure` 执行，通常用于声明超参数
- `structure`：网络结构主脚本

脚本用单引号括起，支持多行字符串：

```ini
[cifa]
parameter = '
    batch = 100;
    lr    = 1e-3;
'
structure = '
    X = Matrix(28, 28, 1, batch);
    ...
    setXY(X, A);
    addLoss(crossEntropy(A, getY()));
'
```

脚本中可以用 `section::key` 的方式引用 INI 其他段的值：

```c++
batch = train::batch;
lr    = train::learn_rate_base;
```

cifa 支持 C 风格的 `auto` 变量声明、`for`/`if` 控制流以及 `{v1, v2, ...}` 列表字面量：

```c++
auto batch = 100;
auto act   = active_relu;
Y = maxPool(X, {2, 2});   // {2,2} 即 [2,2] 列表
for (i = 0; i < 10; i++) { ... }
```

## 二、矩阵创建

| 函数 | 说明 |
|------|------|
| `Matrix(w, h, c, n)` | 创建矩阵，参数可为 1–4 个维度值（w、h、c、n），不足的维度默认为 1 |
| `M(w, h, c, n)` | 同上，`Matrix` 的简写 |
| `MatrixF(w, h, c, n)` | 始终创建 float32 矩阵（即使全局 `data_type=half`），用于 token id 等不能用 half 的矩阵 |
| `MatrixWithName("name", w, h, c, n)` | 创建矩阵并同时设置权重名（等价于 `W=Matrix(...); setWeightName(W,"name")`） |

矩阵的四个维度依次为：**宽(w) × 高(h) × 通道(c) × 批次(n)**。

> 通过 `Matrix()` 创建的矩阵默认允许反向传播和参数加载。  
> 框架会根据计算图拓扑自动识别权重矩阵（不连接 X、但连接到损失函数的矩阵），通常无需手动指定。

```c++
X  = Matrix(28, 28, 1, batch);   // 输入：28×28 灰度图，batch 张
W1 = Matrix(5, 5, 1, 32);        // 卷积核：5×5，1 输入通道，32 输出通道
b1 = Matrix(1, 1, 32, 1);        // 偏置：每通道 1 个标量
Wfc = Matrix(256, 800);          // 全连接权重：256 输出，800 输入（参数可少于 4 个）
```

## 三、基本运算符

矩阵对象支持直接使用运算符：

| 运算符 | 含义 | 自动判断 |
|--------|------|----------|
| `A + B` | 矩阵加法 | 若 B 的 number=1 且尺寸与 A 不同，自动视为加偏置（addBias） |
| `A * B` | 矩阵乘法（gemm） | — |
| `A * k` / `k * A` | 标量缩放 | k 为数值 |

## 四、网络层函数

### 卷积 / 相关

```c++
Y = conv(X, W)                    // 卷积，步长默认 {1,1}，填充默认 {0,0}
Y = conv(X, W, stride, padding)   // 指定步长和填充，例如 {1,1}、{0,0}
Y = corr(X, W, stride, padding)   // 归一化互相关（NCC）：
                                  // 先将 W 零均值化并归一化，再对 X 每个局部块做同样处理，
                                  // 最后计算归一化点积，输出值域 [-1, 1]
                                  // 主要用于模板匹配，而非常规特征提取
Y = deconv(X, W)                  // 转置卷积（反卷积），步长/填充参数同 conv
Y = deconv(X, W, stride, padding) // W: (kW, kH, C_out, C_in)，输出通道数=C_out
                                  // 输出尺寸: (W_in-1)*stride+kW-2*pad
```

> 卷积算法由 INI `[train]` 段的 `conv_algo` 键控制（默认 -1 自动选择），不在脚本中指定。

### 池化

```c++
Y = maxPool(X, window)                       // 最大池化
Y = maxPool(X, window, stride, padding)
Y = averagePool(X, window)                   // 平均池化（不含 padding）
Y = averagePool(X, window, stride, padding)
Y = reversePool(X, window)                   // 反池化（平均反池化）
Y = reversePool(X, window, stride, padding)
Y = pool(X, pool_type, window, stride, padding)  // 通用池化，pool_type 见下
Y = poolChannel(X, pool_type)                // 在通道维度上池化
Y = upsample(X, sh, sw)                      // 空间上采样：H×sh, W×sw（最近邻）
Y = upsample(X, sh, sw, bilinear)            // bilinear=1 使用双线性插值，=0 最近邻
                                             // sh=sw 时可只写一个参数：upsample(X, s)
```

`pool_type` 常量（可用于 `pool()` 第二参数，同时也注册为数值常量）：

| 常量 | 说明 |
|------|------|
| `pool_max` | 最大池化 |
| `pool_average` | 平均池化（不含 padding，`pool_average_no_padding` 的别名） |
| `pool_average_no_padding` | 平均池化（不含 padding） |
| `pool_average_padding` | 平均池化（含 padding） |
| `pool_ma` | 实验性池化 |

### 激活函数

```c++
Y = active(X, act_type)              // 通用激活，act_type 见下（大小写不敏感）
Y = active(X, act_type, int_params)  // 含整数参数（如 clipped_relu 的上限）
// 各激活函数也可直接按名称调用：
Y = relu(X)               Y = sigmoid(X)
Y = tanh(X)               Y = softmax(X)
Y = softmax_fast(X)       Y = softmax_log(X)
Y = sigmoid_ce(X)         Y = softmax_ce(X)    // CE：反向误差直通，用于多出口网络
Y = softmax_fast_ce(X)    Y = elu(X)
Y = selu(X)               Y = silu(X)
Y = leaky_relu(X)         Y = dropout(X)
Y = abs(X)                Y = square(X)
Y = sin(X)                Y = zigzag(X)
Y = sum_max(X)            Y = zero_channel(X)
Y = softplus(X)           // 仅 CPU
Y = softmaxChannel(X)     // 沿 width 维（序列维）做 softmax，用于注意力分数归一化
Y = gelu(X)               // Gaussian Error Linear Unit（tanh 近似版）
```

`act_type` 常量完整列表（由 Option 注册，脚本中通过 `active_<name>` 引用，数值仅供参考）：

| 常量 | 值 | 含义 |
|------|----|------|
| `active_none` | -1 | 无激活 |
| `active_sigmoid` | 0 | Sigmoid |
| `active_relu` | 1 | ReLU |
| `active_tanh` | 2 | Tanh |
| `active_clipped_relu` | 3 | Clipped ReLU |
| `active_elu` | 4 | ELU（仅 GPU） |
| `active_softmax` | 5 | Softmax |
| `active_softmax_fast` | 6 | Softmax Fast |
| `active_softmax_log` | 7 | Softmax Log |
| `active_dropout` | 9 | Dropout |
| `active_softplus` | 11 | Softplus（仅 CPU） |
| `active_sigmoid_ce` | 20 | Sigmoid + CE 直通 |
| `active_softmax_ce` | 21 | Softmax + CE 直通 |
| `active_softmax_fast_ce` | 22 | Softmax Fast + CE 直通 |
| `active_sin` | 23 | Sin |
| `active_zigzag` | 24 | Zigzag |
| `active_leaky_relu` | 26 | Leaky ReLU（别名 `active_lrelu`） |
| `active_selu` | 27 | SELU |
| `active_abs` | 28 | Abs |
| `active_silu` | 30 | SiLU |
| `active_gelu` | 36 | GELU（tanh 近似，GPU/CPU 均支持） |

### 其他层

```c++
Y = add(A, B)                  // 矩阵加法（等价于 A + B）
Y = add([A, B, C], [w1,w2,w3]) // 多输入加权求和
Y = addBias(X, b)              // 显式加偏置
Y = elementMul(A, B)           // 逐元素乘法
Y = mul(A, B)                  // 矩阵乘法（可指定维度参数）
Y = concat([A, B, C])          // 通道维拼接
Y = reshape(X, dim)            // 改变形状，dim 为整数列表；总元素数须一致，-1 可自动推断
Y = reshapeBatch(X, dim)       // 同 reshape，但允许改变 batch 维（number），总元素数仍须一致
Y = max(A, B)                  // 逐元素取较大值
Y = scale(X, k)                // 标量缩放（等价于 X * k）
Y = roiAlign(feat, boxes, roi_size)                        // ROI Align（spatial_scale 默认 1.0）
Y = roiAlign(feat, boxes, roi_size, spatial_scale)         // 带显式 spatial_scale
// feat: (W,H,C,B)；boxes: (4,N,1,B) 像素坐标 (x1,y1,x2,y2)；输出: (roi_size,roi_size,C,N*B)
```

### 归一化

```c++
Y = layerNorm(X, W, b)    // 层归一化：对通道维归一化后仿射变换
                           // W, b 形状均为 (1,1,c,1)，与 X 通道数相同
Y = groupNorm(X, G, scale, bias)           // Group Normalization，G 为组数（C 必须整除 G）
Y = groupNorm(X, G, scale, bias, epsilon)  // 默认 epsilon=1e-5
                           // 每组对 W×H×C/G 个元素归一化，scale/bias 为逐通道可训练参数
                           // 常用于生成模型（Diffusion、GAN）等 batch_size 较小的场景
```

### Batched 矩阵乘法

```c++
Y = batchedMul(A, B, M, N, K, batch)          // 每个 batch slice 做 GEMM：M×K × K×N → M×N
Y = batchedMul(A, B, M, N, K, batch, ta, tb)  // ta/tb=1 表示对应矩阵转置
```

CUDA 路径调用 `cublasSgemmStridedBatched`，HIP 路径类似。矩阵以列优先存储（与 cuBLAS 约定一致）。

### 自注意力

```c++
Y = attention(Q, K, V, dk)               // 缩放点积注意力
Y = attention(Q, K, V, dk, causal)       // causal=1 启用因果掩码
Y = attention(Q, K, V, dk, causal, bias) // 额外加性偏置（softmax 之前），用于 ALiBi 等相对位置编码
```

计算流程：$K^T \cdot Q \;\to\; /\sqrt{d_k} \;[+\text{bias}]\;\to\; \text{softmax}_{channel} \;\to\; V \cdot \text{attn}$

其中 softmax 在通道维（token 维）执行。内部通过 `batchedMul` 实现，Q/K/V 通常由 1×1 卷积投影生成：

```c++
// 假设输入 N1: (D, T, 1, batch)，T 为 token 数，D 为 embedding 维度
Q = conv(N1, Matrix(1, 1, D, D))    // 1×1 conv，token-wise 线性投影
K = conv(N1, Matrix(1, 1, D, D))
V = conv(N1, Matrix(1, 1, D, D))
N2 = attention(Q, K, V, D)          // 自注意力（无因果掩码）
// 带 ALiBi 偏置示例（bias 形状须与 scores 可广播）
bias = Matrix(T, T, 1, 1)           // 形状 (T_k, T_q, 1, 1) 或 (T_k, T_q, H, B)
N2 = attention(Q, K, V, D, 0, bias)
```

### Token 操作

```c++
Y = prependToken(X, cls)    // 在序列头部插入可学习 cls token
                             // X: (D,T,1,B) → Y: (D,T+1,1,B)，cls: (D,1,1,1)
Y = firstToken(X)           // 提取序列第一个 token
                             // X: (D,T,1,B) → Y: (D,1,1,B)
```

`prependToken` + `firstToken` 配合使用可实现标准 ViT 的 CLS token 分类头：在序列前添加 cls token，经过 Transformer 块后提取第一个 token 用于分类。

## 六、损失函数

```c++
addLoss(crossEntropy(A, getY()))           // 交叉熵损失
addLoss(crossEntropy(A, getY(), weight))   // 带权重的交叉熵
addLoss(focal(A, getY()))                  // Focal Loss
addLoss(L2(W))                             // L2 正则化（权重衰减）
addLoss(mseLoss(A, getY()))                // MSE 重建损失：mean((A-Y)²)，常用于自编码器
addLoss(l1Loss(A, getY()))                 // L1 重建损失：mean(|A-Y|)
addLoss(klLoss(mu, log_var))               // KL 散度：-0.5·mean(1+lv-μ²-exp(lv))，先验 N(0,1)
addLoss(klLoss(mu, log_var, beta))         // β-VAE：KL 损失乘以系数 β
addLoss(0.5 * crossEntropy(A, getY()) + 0.5 * L2(W))  // 多损失加权组合
```

**VAE 重建损失组合示例**：
```c++
// 典型 β-VAE：重建损失 + KL 散度
addLoss(mseLoss(recon, getY()));
addLoss(klLoss(mu, log_var, 0.001));
```

最后一层是 softmax_ce 或 sigmoid_ce 时，框架会自动将该层输出作为预测值参与损失计算，无需再写一次 `crossEntropy`。例如：

如果最后一层是 softmax 或 sigmoid 但没有_ce后缀，默认会出现自动修复，见MatrixOp中的相关部分。如果你明确不想要自动修复（例如想用 sigmoid 输出接L2损失），应自己修改一下代码。


## 七、网络输入输出绑定

```c++
setXY(X, A);   // 指定输入矩阵 X 和网络预测输出矩阵 A（必须调用）
setXA(X, A);   // 同上（别名）
```

> `setXY(X, A)` 中的第二个参数 `A` 是**网络预测输出**（计算图末端激活的结果），而非标签。

获取外部传入矩阵：

```c++
X = getX()             // 获取输入矩阵（由数据准备器填充，对应 setXY 的第一个参数）
A = getA()             // 获取网络预测输出矩阵（对应 setXY 的第二个参数，用于外部读取结果）
Y = getY()             // 获取标签矩阵（由数据准备器在训练时填充的真实标签，用于计算损失）
lw = getLossWeight()   // 获取损失权重矩阵（由 setLossWeight 设置）
m  = getMatrix("name") // 获取由准备器传入的命名矩阵
ok = haveMatrix("name")// 判断该矩阵是否存在
showMatrix("name", M)   // 将矩阵注册为可被外部访问的命名矩阵
registerMatrix("name", M) // 同上，showMatrix 的别名
```

> 在计算损失时，通常写 `crossEntropy(A, getY())`：`A` 是预测，`getY()` 是训练标签。

## 八、权重管理

```c++
addWeight(W1, W2, ...)   // 将矩阵提前加入权重列表，使其跳过 initWeights 随机重初始化（保留原始值）
clearWeight()            // 清空手动添加的条目（不影响自动识别逻辑）
setWeightName(W, "name") // 为权重矩阵设置名字，配合 named_weights=1 时按名字加载预训练权重
setIsWeight(M, 1)        // 强制设置矩阵的权重标志：1=是权重，0=不是权重
                         // 主要用于将中间结果正确标记为非权重（防止被误识别为可训练参数）
```

> **正常情况下无需调用 `addWeight()`**。  
> 框架在 `initWeights()` 中会自动识别所有结构性权重并进行随机初始化。  
> `addWeight(W)` 的主要用途是：将矩阵提前写入权重列表，这样 `initWeights()` 会跳过对它的随机重初始化（例如已通过 `setValue` 手动赋值，或将要从已保存模型加载的矩阵）。  
> 最终 `weights_` 列表中会同时包含手动添加和自动识别的所有权重，一起由 Solver 更新并进行模型保存/读取。

## 十一、语言模型算子（LLM）

以下算子专为大语言模型（Transformer Decoder）推理设计。

### 词表嵌入

```c++
Y = embed(ids, W)
// ids: token id 矩阵，形状 (T, 1, 1, B)，元素为整数（float 截断取整）
// W:   词表权重矩阵，形状 (D, 1, 1, V)，V 为词表大小
// Y:   输出嵌入，形状 (D, T, 1, B)
```

### RMS 归一化

```c++
Y = rmsNorm(X, scale)
// 对通道维做 RMS 归一化（无均值中心化），再用 scale 仿射缩放
// scale 形状 (D, 1)，与 X 通道数相同
```

### 旋转位置编码（RoPE）

```c++
// 预计算位置编码表（全层共用，只需调用一次）
cos_tab = ropeCosTbl(T, d_head)            // 默认 base=10000
cos_tab = ropeCosTbl(T, d_head, base)      // Qwen3 使用 base=1000000
sin_tab = ropeSinTbl(T, d_head, base)

// YaRN（NTK-by-parts）：将上下文长度从 original_max 扩展到 T
// scale = T / original_max；beta_fast/beta_slow 默认 32/1
cos_tab = ropeCosTblYaRN(T, d_head, base, scale, original_max)
sin_tab = ropeSinTblYaRN(T, d_head, base, scale, original_max)
// 可选精细控制：
cos_tab = ropeCosTblYaRN(T, d_head, base, scale, original_max, beta_fast, beta_slow)

// 对 Q 或 K 施加旋转位置编码（Qwen 风格 half-rotate）
Y = rope(X, cos_tab, sin_tab)
// X:       (dh, T, H, B)
// cos/sin: (dh/2, T, 1, 1)
// rope() 调用不变，YaRN 完全通过预计算表实现，不修改推理图
```

**YaRN 参数说明**

| 参数 | 说明 |
|------|------|
| `scale` | 扩展倍数 = 目标上下文 / 原始训练长度 |
| `original_max` | 模型训练时的最大序列长度 |
| `beta_fast` | 高频边界（旋转次数），默认 32 |
| `beta_slow` | 低频边界（旋转次数），默认 1 |

**各维度行为**（波长 λ = 2π / θ_d）：
- `λ < original_max / beta_fast`（高频）：不缩放，捕捉局部模式
- `λ > original_max / beta_slow`（低频）：线性插值，位置坐标除以 scale
- 中间区域：平滑混合

**用法示例**（将 Qwen3-0.5B 从 32768 扩展到 131072）：
```c++
int scale = 4;
int orig  = 32768;
cos_tab = ropeCosTblYaRN(T, dh, 1000000, scale, orig)
sin_tab = ropeSinTblYaRN(T, dh, 1000000, scale, orig)
```

### KV 缓存

```c++
Y = kvcache(X_new, cache)
// X_new: 本步新增的 K 或 V，形状 (dh, t, H, B)，t 通常为 1（单步推理）
// cache: 预分配的全序列缓冲矩阵，形状 (dh, T_max, H, B)
// Y:     与 cache 共享数据，形状为 (dh, T_max, H, B)，仅前 cur_len 列有效
// 每轮对话前须调用 Net::resetKVCache() 将写入位置归零
```

### 轴置换与形状变换

```c++
Y = permute(X, {p0, p1, p2, p3})      // 4 维任意轴置换，pN 为新第 N 维对应的原始轴编号
Y = reshape(X, {d0, d1, d2, d3})      // 改变形状，总元素数须一致，-1 可自动推断一个维度
Y = reshapeBatch(X, {d0, d1, d2, d3}) // 同 reshape，但允许改变 batch（number）维
                                       // 常用于 GQA 中将 (dh, T, Hkv, B) 重解释为 (dh, T, 1, Hkv*B)
```

### GQA 头扩展（tile）

```c++
Y = tile(X, {r0, r1, r2, r3})
// 沿各轴重复整数倍，用于 GQA 将 KV 头数从 Hkv 扩展到 H
// 例：tile(K, {1, 1, 2, 1}) 将通道维重复 2 次（Hkv×2 = H）
```

### 因果注意力

```c++
Y = attention(Q, K, V, dk)               // 非因果（训练时双向注意力）
Y = attention(Q, K, V, dk, causal)       // causal=1 启用下三角因果掩码（自回归推理必须设为 1）
Y = attention(Q, K, V, dk, causal, bias) // 附加加性偏置（softmax 前）：用于 ALiBi 等相对位置编码
// Q/K/V: (dh, T, H, B)
// dk:    缩放因子（通常为头维度 dh）
// bias:  形状须与 scores (T_k, T_q, H, B) 可广播，不传则无偏置
// 计算：softmax((K^T·Q / sqrt(dk)) + bias) · V
```

### VAE 重参数化

```c++
z = reparam(mu, log_var)
// 变分自编码器的重参数化技巧
// mu, log_var: 形状 (D,1,1,B)，由编码器输出的均值和对数方差
// 训练时：z = mu + exp(log_var*0.5) * ε，ε~N(0,1)（中间噪声存入 workspace 供反向使用）
// 推理时：z = mu（确定性输出）
```

配合 `klLoss` 使用构成标准 VAE：

```c++
mu      = conv(enc, W_mu);
log_var = conv(enc, W_lv);
z       = reparam(mu, log_var);
dec_out = decoder(z);
addLoss(mseLoss(dec_out, getY()));
addLoss(klLoss(mu, log_var, 0.001));
```

### PixelShuffle 上采样

```c++
Y = pixelShuffle(X, r)
// 空间维度上采样 r 倍，通道数缩小 r² 倍
// X: (r²·C, H, W, B)  →  Y: (C, r·H, r·W, B)
```

## 十四、示例：Qwen3-0.5B（大语言模型推理）

Qwen3-0.5B 是阿里通义千问系列的小型语言模型，结构为标准 Transformer Decoder，并加入了每头独立 QK 归一化（Qwen3 特有）。完整模型共 28 层，以下仅展示单层写法，实际配置需重复 28 次并分别命名权重矩阵和 KV 缓存。

主要参数：

| 参数 | 值 |
|------|----|
| 词表大小 V | 151936 |
| 隐层维度 D | 896 |
| 注意力头数 H | 16 |
| KV 头数 Hkv（GQA）| 8 |
| 头维度 dh | 56 |
| FFN 维度 F | 4864 |
| 层数 | 28 |
| 最大序列长度 T | 512 |
| RoPE base | 1000000 |

```ini
[train]
T             = 512
batch         = 1
dim           = 896
n_heads       = 16
n_kv_heads    = 8
d_head        = 56
ffn_dim       = 4864
vocab         = 151936
rope_base     = 1000000
load_file     = qwen3/qwen3_decoder_cccc.bin
named_weights = 1
need_free_mem = 0
output_log    = 0

[cifa]
structure='
    T   = train::T;
    B   = train::batch;
    D   = train::dim;        // 896
    H   = train::n_heads;    // 16
    Hkv = train::n_kv_heads; // 8
    dh  = train::d_head;     // 56
    F   = train::ffn_dim;    // 4864
    V   = train::vocab;      // 151936

    // 输入 token id 序列 (T, 1, 1, B)
    X  = Matrix(T, 1, 1, B);

    // 词表嵌入：W_E (D,1,1,V) -> H0 (D,T,1,B)
    H0 = embed(X, Matrix(D, 1, 1, V));

    // RoPE 位置编码表（全层共用）
    cos_tab = ropeCosTbl(T, dh, train::rope_base);
    sin_tab = ropeSinTbl(T, dh, train::rope_base);

    // KV 缓存（每层单独分配，此处仅展示第 0 层）
    Kcache0 = Matrix(dh, T, Hkv, B);
    Vcache0 = Matrix(dh, T, Hkv, B);
    // ... Kcache1~Kcache27, Vcache1~Vcache27 此处省略

    // ===== Transformer Layer 0 =====

    // Pre-attention RMSNorm
    H0n  = rmsNorm(H0, Matrix(D, 1));

    // Q/K/V 线性投影（无偏置）
    Qraw = reshape(Matrix(H*dh,   D) * H0n, {dh, T, H,   B});
    Kraw = reshape(Matrix(Hkv*dh, D) * H0n, {dh, T, Hkv, B});
    Vraw = reshape(Matrix(Hkv*dh, D) * H0n, {dh, T, Hkv, B});

    // 每头独立 QK 归一化（Qwen3 特有）
    Qh = rmsNorm(Qraw, Matrix(dh, 1));
    Kh = rmsNorm(Kraw, Matrix(dh, 1));

    // 旋转位置编码
    Q = rope(Qh, cos_tab, sin_tab);
    K = rope(Kh, cos_tab, sin_tab);

    // 写入 KV 缓存，输出含历史序列的 K/V
    Kc = kvcache(K,    Kcache0);
    Vc = kvcache(Vraw, Vcache0);

    // GQA：将 KV 头数从 Hkv=8 扩展到 H=16（每组重复 2 次）
    Kexp = tile(Kc, {1, 1, 2, 1});
    Vexp = tile(Vc, {1, 1, 2, 1});

    // 因果缩放点积注意力
    Ao = attention(Q, Kexp, Vexp, dh, 1);

    // 输出投影 + 残差
    Aout = Matrix(D, H*dh) * reshape(Ao, {H*dh, T, 1, B});
    R1   = H0 + Aout;

    // SwiGLU FFN：Post-FFN RMSNorm + Gate/Up 投影 + 残差
    R1n  = rmsNorm(R1, Matrix(D, 1));
    Gate = active(Matrix(F, D) * R1n, active_silu);
    Up   = Matrix(F, D) * R1n;
    FFN  = Matrix(D, F) * elementmul(Gate, Up);
    R2   = R1 + FFN;

    // ...（第 1-27 层与第 0 层相同，权重矩阵和缓存名称不同，此处省略）

    // 最终 RMSNorm + LM 头（logits）
    Hf = rmsNorm(R2, Matrix(D, 1));
    A  = Matrix(V, D) * Hf;    // (V, T, 1, B)

    setXY(X, A);
'
```
