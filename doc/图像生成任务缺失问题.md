## 已解决问题

| # | 问题 | 解决方案 | 新增算子 |
|---|------|----------|----------|
| 1 | 转置卷积（Transposed Convolution） | CUDA 路径调用 `cudnnConvolutionBackwardData`；反向调用 `convolutionForward` + `cudnnConvolutionBackwardFilter` | `deconv(A, W [, {stride}, {padding}])` |
| 2 | GroupNorm | LayerNorm 内核（`cuda_layer_norm_fwd`）归一化后接 `cuda_group_norm_affine_fwd` 施加 per-channel scale/bias；6 个 workspace 槽（mean/invstd/X_hat/dscale/dbias/dX_hat） | `groupNorm(X, G, scale, bias [, epsilon])` |
| 3 | VAE 重参数化 | CPU Box-Muller 采样 eps → 上传到与 mu 同设备的 Matrix；训练时 z=mu+exp(log_var/2)*eps，推理时 z=mu；eps 存于 workspace[0] 供反向使用 | `reparam(mu, log_var)` |
| 4 | KL 散度损失 | `-0.5*mean(1+lv-mu²-exp(lv))`；`backwardLoss` 实现 d_mu=mu/N*scale、d_lv=0.5*(exp(lv)-1)/N*scale；支持 β-VAE：`addLoss(klLoss(mu,lv)*beta)` | `klLoss(mu, log_var [, beta])` |
| 5 | MSE / L1 重建损失 | MSE 梯度 `2*(A-Y)/N`；L1 梯度 `sign(A-Y)/N`；均复用 `Matrix::add` 或 `cuda_l1_bwd` 内核 | `mseLoss(A, Y)`, `l1Loss(A, Y)` |
| 6 | 双线性/最近邻上采样 | `X(W,H,C,N)→Y(W*sw,H*sh,C,N)`；CUDA 有独立 nearest/bilinear 内核；bilinear 反向用 atomicAdd；align_corners=False | `upsample(X, sh, sw [, bilinear])` |

## 详细说明

### 问题 1：转置卷积（Transposed Convolution）

**影响**：VAE decoder 和 GAN generator 需要逐步将低分辨率特征图放大回原始尺寸；U-Net decoder 路径也需要。cuDNN 原生提供 `cudnnConvolutionBackwardData`（即转置卷积），cccc 的 `MatrixEx::convolutionBackwardDX` 已在内部调用，但未作为独立算子暴露给 cifa 脚本。

**期望实现**：
```c++
B = deconv(A, W)               // 等价于 conv 的反向数据路径
B = deconv(A, W, {2,2}, {1,1}) // stride=2 实现 2× 上采样
```
底层直接复用 `MatrixEx::convolutionBackwardDX` 和 `convolutionBackwardDW`，反向则调用 `convolutionForward`，两者互为转置关系。

### 问题 2：GroupNorm

**影响**：扩散模型（Stable Diffusion、DDPM 等）的 U-Net 将 BatchNorm 替换为 GroupNorm，原因是训练时 batch size 通常为 1~2，BatchNorm 的统计估计方差极大；GroupNorm 只在通道维内分组归一化，不依赖 batch size。

**期望实现**：
```c++
B = groupNorm(A, groups, scale, bias)
// A 形状 (W, H, C, N)，沿 C 分为 groups 组，每组内 (W*H*C/groups) 元素一起归一化
// scale/bias 形状 (C,)
```
cuDNN 4.x 之后没有直接的 GroupNorm 接口；可用已有的 LayerNorm 内核改写（将 C 维分组后逐组归一化），或用 CUDA 自定义内核（置于 `cccc-cuda`）。

### 问题 3：VAE 重参数化算子

**影响**：VAE 编码器输出均值 μ 和对数方差 log σ²，通过重参数化技巧 $z = \mu + \sigma \cdot \varepsilon$（$\varepsilon \sim \mathcal{N}(0,1)$）采样潜在向量，使梯度可以回传到 μ 和 log σ²。前向需要随机采样，反向梯度对 μ 为恒等、对 log σ² 为 $\frac{1}{2}\varepsilon \cdot \sigma$。

**期望实现**：
```c++
z = reparam(mu, log_var)
// 前向：随机生成 eps ~ N(0,1)，z = mu + exp(log_var/2) * eps
// 内部保存 eps 供反向使用
```
需要在 `cccc-cuda`（或 CPU 分支）中实现随机高斯采样，并将 eps 存入 workspace。训练/推理模式需要区分（推理时直接取 mu）。

### 问题 4：KL 散度损失

**影响**：VAE 损失 = 重建损失 + β·KL 散度。KL 项为：
$$
\mathcal{L}_{KL} = -\frac{1}{2} \sum \left(1 + \log\sigma^2 - \mu^2 - \sigma^2\right)
$$
其梯度对 μ 为 μ，对 log σ² 为 $\frac{1}{2}(\sigma^2 - 1)$。

**期望实现**：
```c++
addLoss(klLoss(mu, log_var))          // 标准 KL，先验 N(0,1)
addLoss(klLoss(mu, log_var, beta))    // β-VAE，缩放系数
```
在 `MatrixOp` 中新增 `KL_LOSS` 类型，前向计算 KL 值（用于监控），反向直接写入 mu.d() 和 log_var.d()。

### 问题 5：MSE / L1 重建损失

**影响**：图像重建任务常用像素级均方误差（MSE）或 L1 损失，而不是分类用的交叉熵。框架现有 `crossEntropy(A, Y)` 和 `focal`；`L2(A)` 是权重范数正则化，反向也标注为"未完成"。

**期望实现**：
```c++
addLoss(mseLoss(A, Y))    // 均方误差：mean((A-Y)²)，梯度 2(A-Y)/N
addLoss(l1Loss(A, Y))     // 平均绝对误差：mean(|A-Y|)，梯度 sign(A-Y)/N
```
变通方案：`A - Y` 可用现有减法构造差矩阵，再用 `ACTIVE_FUNCTION_SQUARE` 平方后做全局 pool，但反向路径较长，效率低；专用算子更直接。

### 问题 6：双线性 / 最近邻上采样

**影响**：Diffusion U-Net 的上采样路径常采用"最近邻上采样 + 普通卷积"替代转置卷积，可以避免棋盘纹伪影（checkerboard artifacts）。GAN 的 Progressive Growing 也用到逐步上采样。

框架已有 `pixelShuffle`（亚像素卷积），但它是 channel-to-space 重排，不是空间插值；`reversePool` 是最大反池化（位置驱动），也不是双线性插值。

**期望实现**：
```c++
B = upsample(A, {2, 2})              // 最近邻，放大 2× (H, W 各)
B = upsample(A, {2, 2}, "bilinear")  // 双线性插值
```
反向：最近邻上采样反向等价于 averagePool + 缩放；双线性反向为双线性反插值。可用 cuDNN 的空间变换算子或自定义 CUDA 内核（置于 `cccc-cuda`）实现。

### 问题 7：时间步嵌入（Sinusoidal Time Embedding）

**影响**：DDPM 等扩散模型需要将离散时间步 $t \in \{0, 1, \ldots, T-1\}$ 编码为连续嵌入向量注入 U-Net，通常用正弦位置编码：
$$
\text{emb}(t)_i = \begin{cases} \sin(t / 10000^{2i/d}) & i < d/2 \\ \cos(t / 10000^{2(i-d/2)/d}) & i \ge d/2 \end{cases}
$$
之后经两层线性+SiLU投影。

框架已有 `ACTIVE_FUNCTION_SIN`，但缺少批量整数 t → d 维嵌入的一步算子；标量 t 输入时需要为每个样本单独构造，无法向量化。

**期望实现**：
```c++
emb = sinTimeEmbed(t, d, base)
// t: (1,1,1,B) float-as-int，d: 嵌入维度，base: 默认10000
// 输出: (d, 1, 1, B)，无可训练参数（固定正弦编码）
// 之后接两层 linear + silu 做可学习投影
```
无反向需求（固定编码），实现相对简单。

### 问题 8：扩散噪声调度（DataPreparer 层面）

**影响**：Diffusion 模型训练的数据流为：给定原图 $x_0$ 和随机时间步 $t$，按噪声调度计算 $x_t = \sqrt{\bar\alpha_t} x_0 + \sqrt{1-\bar\alpha_t} \varepsilon$（$\varepsilon \sim \mathcal{N}(0,1)$），网络输入 $(x_t, t)$，目标为预测 $\varepsilon$。这一加噪过程在 `DataPreparer` 阶段完成。

**期望实现**：新增 `DataPreparerDiffusion` 子类，或在 `DataPreparerImage` 中增加 `noise_schedule` 选项：
```ini
[data_preparer]
type = image
noise_schedule = cosine      ; linear | cosine | sigmoid
diffusion_T   = 1000
diffusion_s   = 0.008        ; cosine offset
output_noise  = 1            ; Y 输出原始噪声 ε
```
该类需要：(1) 预计算 $\bar\alpha_t$ 表；(2) 每个样本随机采样 $t$；(3) 生成 $(x_t, t)$ 作为 X，$\varepsilon$ 作为 Y。

### 问题 9：多网络交替训练（GAN 工作流）

**影响**：GAN 训练需要在同一 epoch 内交替优化判别器 D 和生成器 G，且两者的 loss 计算互相依赖（G 的输出是 D 的输入，G 的 loss 来自 D 的输出）。当前 `MainProcess` 仅支持单网络顺序训练循环。

**变通方案（现阶段）**：
- 将 G 和 D 定义在同一个 cifa 脚本中，作为两个子图
- 用 `setNeedBack` / `setNeedBackState` 控制哪段子图参与反向
- 在两次 `active()` 调用之间手动切换梯度状态

**理想实现**：在 `MainProcess` 层面支持 `[net_G]` + `[net_D]` 双网络配置，工作流按轮次自动交替调用各自的前向/反向/更新流程。工程量较大，属于框架级改动。

## z-image-turbo 模型分析（nihui-szyl/z-image-ncnn）

> 记录于 2025-05。来源：HuggingFace `nihui-szyl/z-image-ncnn` 仓库 `z-image-turbo/` 目录。

### 架构概述

z-image-turbo 是基于 **FLUX.1 类双流 DiT（Diffusion Transformer）** 的图像生成模型 turbo 蒸馏版。
整体结构将文本-图像双流 Transformer 拆分为多个独立 ncnn 子模型，支持按需加载和流水线推理。

```
文本 prompt
    │
    ▼
text_encoder (7.84 GB)    ← T5/LLaMA 大型文本编码器，输出 2560-dim token 序列
    │
    ▼
cap_embedder              ← RMSNorm(2560) + Gemm 2560→3840，适配 DiT 隐空间
    │
    ├─────────────────────────────────────────────────┐
    │ 图像侧                                           │ 文本侧
    ▼                                                 ▼
x_embedder                                    (text tokens 3840-dim)
  (Gemm 64→3840)
  latent patch 嵌入
    │
    │    时间步 t
    │        │
    │        ▼
    │   t_embedder                ← 正弦编码(128) → concat → Gemm→SiLU→Gemm，输出 256-dim
    │
    ├── context_refiner (708 MB)  ← 2× 双流 Transformer（RMSNorm+RoPE+SDPA+SiLU）
    │       对图像 token 和文本 token 同时精炼
    │
    ├── noise_refiner  (724 MB)   ← 2× 单流 Transformer，含 adaLN 时间调制
    │       Slice/chunk: Gemm(256→15360) → 4×3840 scale/shift/gate
    │
    └── unified        (10.9 GB) ← 主干 DiT（全部 Transformer 块，单流+双流混合）
            │
            ▼
        all_final_layer           ← LayerNorm + SiLU gate + Gemm 3840→64，输出 patch logits
            │
            ▼（DDIM 去噪循环结束后）
          vae (99.2 MB)           ← Conv+GroupNorm+Swish+Attention+Upsample_nearest，解码图像
            │
            ▼
        RGB 图像输出
```

### 各子模型详细说明

| 模型文件 | 大小 | 用途 | cccc 算子状态 |
|---------|------|------|--------------|
| `text_encoder` | 7.84 GB | 文本编码器（LLaMA/T5 类 LLM），编码 prompt → 2560-dim token 序列 | 尚未转换，待分析 |
| `transformer_all_x_embedder` | 499 kB | 图像 patch 嵌入：Gemm 64→3840（64 = 2×2 patch × 16ch VAE latent） | ✅ batchedMul |
| `transformer_cap_embedder` | 19.7 MB | caption 适配：RMSNorm(2560) + Gemm 2560→3840 | ✅ rmsNorm + batchedMul |
| `transformer_t_embedder` | 1.05 MB | 时间步嵌入：正弦(128-dim) + concat + Gemm→SiLU→Gemm，输出 256-dim | ⚠️ 问题 #7（sinusoidal） |
| `transformer_context_refiner` | 708 MB | 2× 双流 Transformer（文本+图像同时 RMSNorm+RoPE+SDPA+SiLU gate FFN） | ✅ 全部算子已有 |
| `transformer_noise_refiner` | 724 MB | 2× 单流 Transformer + adaLN 时间调制（Slice/chunk + TanH） | ❌ 问题 #10（Slice） |
| `transformer_unified` | 10.9 GB | 主干 DiT（多个混合双流/单流 Transformer 块） | ❌ 同上 |
| `transformer_all_final_layer` | 2.47 MB | 最终投影：LayerNorm + SiLU gate + Gemm 3840→64 | ✅ 全部算子已有 |
| `vae` | 99.2 MB | VAE 解码器：Conv+GroupNorm+Swish+Attention+Upsample_nearest | ✅ 全部算子已有 |

### 新增缺失问题

#### 问题 10：Slice / chunk（沿轴均分 Tensor）

**出现位置**：`noise_refiner` 和 `unified` 中，时间步嵌入经 `Gemm 256→15360` 后被 `Slice` 均分为 4 份（各 3840-dim），用于 adaptive LayerNorm（adaLN）的 scale/shift/gate 调制：
```
Gemm gemm_0 (256→15360)
Slice chunk_0 : 1→4，axis=1，各分量 3840-dim
    ├── out_0 → add (scale 1)
    ├── out_1 → TanH → mul (gate)
    ├── out_2 → add (scale 2)
    └── out_3 → TanH → mul (gate 2)
```

**期望实现**：
```c++
// 沿 dim 维将 X 均分为 n 份，返回 n 个 Matrix 视图
[A, B, C, D] = chunk(X, n=4, dim=1)
```
底层可实现为 Matrix 的步长视图（无内存拷贝），类似 `concat` 的逆操作。

  - 主干与末层：`net_unified_1036.ini`（float）、`net_final_1036_1024.ini`（float）
- VAE 继续保持 `net_vae_64x64.ini`：`data_type=float` + `disable_activation_memory_reuse=1`。

### 2026-05-13 精度与耗时对比（`Prompt='风的彷徨'`）

这次实际测到的结论不是“ncnn 用更低精度 FP16”，而是：

- ncnn Python 生成脚本 `work/z-image/generate.py` 明确关闭了 `fp16_packed/fp16_storage/fp16_arithmetic`，同时启用 `bf16_packed/bf16_storage`。
- 运行日志也显示 Vulkan 设备支持 `bf16-p/s=1/1`，因此 ncnn 这次实际走的是 **BF16 路径**。
- cccc-sd 则是 **混合精度**：前半段文本/嵌入网络使用 `half`，后半段关键降噪网络与 VAE 使用 `float`。

#### 图像结果差异

两张图不是“细节略有不同”，而是整体构图、服装、背景场景都明显不同：

- cccc-sd 输出更接近海边近景、黑色外套、人物更靠近画面。
- ncnn 输出更接近草地远景、棕色外套、天空占比更大。

像素级对比指标如下：

| 指标 | 数值 |
|------|------|
| shape | `(512, 512, 3)` |
| MAE | `69.1989` |
| RMSE | `87.5810` |
| PSNR | `9.2826 dB` |
| max_diff | `235` |
| identical_pixel_ratio | `0.000000` |

#### 本轮判断

- ncnn 的主要精度差异来自 **BF16**，不是 FP16。
- cccc-sd 的主要性能差异来自 **每个 step 都重新加载 4 个降噪网络**，而非精度模式本身。
- **实测验证（2026-05-13）**：将 4 个降噪网络的加载移到循环外（常驻显存），总耗时从 248s 降至 **69s（2.8× 接近 ncnn 的 42s）**。VRAM 使用情况如下：

  | 加载时间点 | VRAM used | VRAM free |
  |-----------|-----------|-----------|
  | After x_emb | 1,440 MB | 31,167 MB |
  | After noise | 3,378 MB | 29,228 MB |
  | After unified | 21,335 MB | 11,272 MB |
  | After final | 21,508 MB | 11,098 MB |

  4 网合计只占 **~21.5 GB**，未超 32 GB。之前测到 30,441 MB 是因为文本编码器（~9 GB）还未释放时叠加的显存数字，误导了判断。
- 剩余性能差距（69s vs 42s）来自：CUDA vs Vulkan 后端、float vs BF16 精度（float 激活内存更大）、以及 cccc 每网独立重初始化 CUDA context 的开销（每步仍有 `Reset MP device` 开销，因现在是每步复用已加载网络，这部分已消除）。实际每步降噪耗时约 2.5~4.7 s（9 步共约 31 s），与 ncnn 的 4.7 s/step 基本同量级。
